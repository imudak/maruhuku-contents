# Claude Codeが流行っているけど、VS Code + Copilotで困ってないならそのままでいい

Claude Codeが流行っています。

ただ正直に言うと、
自分は **VS Code + GitHub Copilotだけで、今も特に困っていません。**

それでもこの話題が気になったのは、
Claude Codeで「新しい」と言われている体験の多くが、
**自分がすでにCopilotでやっていることと重なって見えた**からです。

---

## 私は「書く前の作業」もCopilotでやっている

よく言われる話として、

- Copilotは「コードを書くためのAI」
- Claude Codeは「書く前の整理をするAI」

という整理があります。

でも、自分の場合は少し違います。

自分は普段から、

- Issueを読んだあと、まずCopilotに相談する
- 既存コードを見せながら「どこが影響しそうか」を一緒に考える
- 修正方針や設計を、書く前に言語化する
- PRに何を書けばいいかも、Copilotと一緒に整理する

といったことを、**VS Codeの中でそのままやっています。**

なので
「Claude Codeでやるとされていることを、もうCopilotでやっている」
という感覚がありました。

---

## それでも Claude Code が評価されている理由は分かる

一方で、Claude Codeが評価されている理由も理解できます。

それは、
**多くの人がCopilotをそこまで使い込んでいない**からだと思います。

一般的なCopilotの使われ方は、

- 書き始めてから補完を出す
- 関数の中で使う
- 今開いているファイルが中心

というものが多い印象です。

この使い方だと、

- 影響範囲の洗い出し
- 設計の整理
- 文章化（Issue / PR）

は、どうしても人がやることになります。

Claude Codeは、
最初から「書く前の整理」をやる前提で設計されている。

だから、
**「Copilotは便利だけど、そこまではやってくれない」**
と感じている人にとって、
Claude Codeはとても分かりやすく見えるのだと思います。


---

## 役割分担がしっくり来なかった理由

Claude CodeとCopilotの話を読んでいて、
自分が一番しっくり来なかったのは
「役割分担」がきれいに切られている説明でした。

Copilotでも、

- 目的を投げる
- 設計を相談する
- 文章を書かせる

ことは普通にできます。

だからこの話は、
**できる・できないの違いではない**と感じています。

違うのは、

- 何を前提に使い始めるか
- どこからAIに頼る設計になっているか

という **入口の違い** です。


---

## 私にとっての Copilot と Claude Code の位置づけ

自分にとっては、

- VS Code + Copilot
  → **考えるところから書くところまで、一緒にやる相棒**

という位置づけです。

Claude Codeは、

- 「書く前の整理」を最初から外に切り出したい人
- IDEを開く前に、まず全体を整理したい人

にとって、
**とても自然な体験を提供しているツール**だと思います。

ただ、自分の場合は
その整理も含めて、すでにCopilotと一緒にやっている。

だから「乗り換える必要がある」とは感じませんでした.


---

## 少しだけ、自分の立場について

ちなみに自分は、
どちらかというと **組み込み寄りの開発者**です。

一方で、

- Webアプリ
- スマホアプリ
- Windowsアプリ

といった開発も並行してやっています。

その立場から見ると、
この話は「分野の違い」ではなく、

> **AIをどこに置くか、どう使うかの話**

だと感じています.


---

## 触った範囲での正直な感想

なお、Claude Codeは自分ではまだ触った程度です。
なので、「Claude Codeの方が明確に優れている」
という具体的な使いどころがあれば、
正直に言って、ぜひ知りたいと思っています。

ただ、少なくとも自分が見た範囲では、
Copilotではできない“能力差”というより、
使い始めの設計思想や導線の違いが、
評価されているように感じました.


---

## 結論

自分にとっては、

- VS Code + Copilotだけで
- 書く前の整理も含めて
- 開発は十分に回っている

というのが正直な感想です。

ただ, Copilotを
補完ツールとしてしか使っていない人にとっては、
Claude Codeの体験はとても魅力的に映ると思います。

だからこれは、
どちらが正しいかの話ではありません。
自分がどう使っているかで、
自然に選べばいい話だと思っています。
