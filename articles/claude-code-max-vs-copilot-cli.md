## はじめに：20倍の価格差は何を買っているのか

2026年2月25日、GitHub Copilot CLIがGA（一般公開）になりました。ターミナルで動くAIコーディングエージェント——Claude Codeの直接的な競合です。

そして価格を見て驚きます。

- **GitHub Copilot Pro**: $10/月
- **Claude Code（Max x20プラン）**: $200/月

20倍。

さらにCopilotは学生無料、GPT-5.3 Codexなら月300プロンプト、思考モードをxhighにしても同カウント。一見すると圧倒的なコスパです。

ではなぜ、私は$200を払い続けているのか。そしてなぜCopilot CLIは「チートレベルのコスパ」なのに、想像ほど話題になっていないのか。

この記事では、両者を**実運用の観点**から比較します。

## 課金モデルの設計思想が根本的に違う

### Copilot：プロンプト単位課金

GitHub Copilotの課金は「**プロンプト1回 = 1 premium request × モデル倍率**」です。

| モデル | 倍率 | Pro（300 req）での実質回数 |
|---|---|---|
| GPT-5 mini / GPT-4.1 | 含まれる | **無制限** |
| GPT-5.3-Codex | 1x | 300回 |
| Claude Sonnet 4.6 | 1x | 300回 |
| Gemini 3 Pro | 1x | 300回 |
| Claude Opus 4.6 | 3x | 100回 |
| o3 / o4系 | 3〜10x | 30〜100回 |

Pro+（$39/月）なら1,500 requestsに増えます。

**メリット**: 1回のプロンプトで思考モードを最大にしても、カウントは変わりません。質問の「深さ」にペナルティがないのは確かに強い。

### Claude Max：トークン上限の使い放題

Claude Maxは月額固定で、上限内なら**何ターンでも、何セッションでも**使えます。1つのタスクで数百ターンの自律実行をしても、追加課金はありません。

**ここが決定的な違いです。**

## 「1プロンプト」の重みが全く違う

この比較で最も見落とされがちなポイントがここです。

### チャットとしての利用

```
ユーザー: 「このエラーの原因は？」
AI: 「〜が原因です」
```

これは1プロンプト。Copilotなら1 request消費。Claude Maxでも微小なトークン消費。**この用途なら、Copilotが圧倒的にコスパが良い。**

### 自律エージェントとしての利用

```
ユーザー: 「認証モジュールにテストを追加して、失敗したら修正して」
AI: ファイル読み取り → テスト生成 → 実行 → 失敗確認 → 修正 → 再実行 → 
    型エラー発見 → 修正 → 再実行 → 全パス確認 → コミット
```

Claude Codeはこれを**1タスクとして自律実行**します。途中のステップ数に制限はありません。

Copilot CLIのAutopilotモードでは、**各ステップがpremium requestとしてカウント**されます。上の例なら10〜15 requests。複雑なリファクタリングなら1タスクで30〜50 requests消費することもあります。

**つまり、Copilot Proの月300回は、自律エージェントとして使うと「10〜30タスク」で枯渇する可能性がある。**

## 並列実行という次元の違い

私の運用では、Claude Codeを**1日に複数セッション並列で走らせています**。

```
09:00 - セッション1: jimucho APIにフィールド追加
09:00 - セッション2: E2Eテスト修正
09:15 - セッション3: フロー定義のリファクタリング
10:00 - cron: 定期データ評価（自動実行）
```

これはClaude Maxの「トークン上限内なら使い放題」だからこそ成り立つワークフローです。

Copilot CLIにも「Background delegation」（`&`プレフィックスでクラウドにオフロード）や「Fleet mode」（並列subagent）がありますが、**すべてpremium requestsを消費します**。Pro+の1,500 requestsでも、この運用だと1週間持ちません。

## Copilot CLIの正当な強み

公平を期して、Copilot CLIが優れている点も挙げます。

### 1. モデルの多様性

Claude Code → Claude一択。Copilot CLI → Claude Opus 4.6、Sonnet 4.6、GPT-5.3-Codex、Gemini 3 Pro、Haiku 4.5から選択可能。タスクに応じて最適なモデルを使い分けられるのは大きな利点です。

### 2. 圧倒的な低価格帯

$10/月でGPT-5.3-Codexが300回使える。コードレビュー、バグの相談、設計の壁打ち——こうした「会話型」の使い方には、これ以上のコスパはありません。

### 3. GitHub統合

Copilot coding agent（CLIとは別機能）はGitHub Issueから直接PRを作成できます。GitHubをワークフローの中心に据えているチームには自然な導入パスです。

### 4. 学生・教育者は無料

GitHub Student Developer Packで**Copilot Proが無料**。学生にとってはこれ一択でしょう。

## 結局、何を買っているのか

| 観点 | Copilot CLI | Claude Code Max |
|---|---|---|
| 月額 | $10〜$39 | $200 |
| 1回の質問のコスパ | ◎ | △ |
| 自律タスクのコスパ | △（ステップ数で消費） | ◎（使い放題） |
| 並列実行 | △（quota消費が加速） | ◎ |
| モデル選択肢 | ◎（5モデル以上） | △（Claude系のみ） |
| 常駐エージェント運用 | ✕（quota制約） | ◎ |
| 学生 | 無料 | $200 |

**$200と$10の差は、「AIに質問する」ツールと「AIを労働力として使う」基盤の差です。**

Copilotは優秀な相談相手。Claude Code Maxは24時間稼働するチームメンバー。必要なのがどちらかで、選択は変わります。

## 誰がどちらを選ぶべきか

**Copilot CLI（$10〜$39/月）が合う人**
- コードの質問やレビューが主な用途
- 1日数回の対話的な利用
- 複数モデルを試したい
- 学生（無料で使える）
- チーム開発でGitHub統合が重要

**Claude Code Max（$100〜$200/月）が合う人**
- AIに「タスクを丸ごと任せたい」
- 1日に複数の自律実行セッションを回す
- CI/cron的な定期自動実行をしている
- 「AIの稼働時間」が直接的に事業価値になる

## おわりに

GitHub Copilot CLIが「評価されていない」と感じる人がいるのは、ある意味で必然です。$10の価格帯のユーザーは、まだAIコーディングへの投資を試行錯誤している段階の人が多い。使いこなすほどに「もっと使いたい」という欲求が生まれ、結局は上位プランや別サービスに移行していく。

一方で、$200を払っている人は「AIなしの開発に戻れない」レベルまで来ている人です。声が大きくなるのは当然です。

**重要なのは、どちらが「良い」かではなく、自分の使い方がどちらの課金モデルに合っているかを理解すること。** 20倍の価格差があるからこそ、この判断は慎重にすべきです。

まずは$10から始めて、300 requestsが何日で尽きるか試してみてください。1ヶ月持つなら、Copilotで十分です。1週間で尽きるなら——$200の世界へようこそ。
