月200ドルの請求書を見て、一瞬だけ手が止まりました。GitHub Copilot Individualが10ドルだと知っていたので、190ドルの差がどこにあるのかを使いながら考えていました。

## 2つのツールの立ち位置

**Claude Code**は、AnthropicのClaude Maxプランに含まれるCLIツールです（Max x5プランで月100ドル、Max x20プランで月200ドル）。ターミナルから`claude`コマンドで起動し、コードの編集・ファイル操作・コマンド実行まで自律的に行います。

**GitHub Copilot CLI**は、GitHub Copilot Individual（月10ドル）に含まれる機能です。エディタ内の補完がメインですが、`gh copilot suggest`や`gh copilot explain`コマンドでターミナル操作の提案・説明もできます。

## 日常的にどちらを使っているか

Claude Codeをメインで使い始めてから、Copilotのターミナル機能はほとんど使わなくなりました。エディタ補完としてのCopilotは継続して使っていますが、CLIとして比べるとClaude Codeに移行しています。

このZennコンテンツリポジトリでの典型的な作業を例に挙げます。

記事を書いた後、こう指示します。

```text
npx textlintを実行してエラーを修正して
```

Claude Codeはそれを受けて順番に実行します。

1. `npx textlint articles/xxx.md`でエラーを確認
2. 各エラーの内容を読んで修正を判断
3. ファイルを直接編集
4. 再度textlintを実行して確認

自分でやる場合は、エラーを確認して該当箇所を開き、修正して保存して再実行という手順を踏みます。Claude Codeはそれを一気にこなします。

Copilot CLIで同じことをしようとすると、`gh copilot suggest "textlintのエラーを修正するには"`でコマンドの提案はもらえますが、実行して結果を確認して判断して修正するという一連の流れは手動です。

## 何が違うのか

整理するとこうなります。

| | Claude Code Max | GitHub Copilot CLI |
|---|---|---|
| 月額 | $100〜$200 | $10（Copilot Individual） |
| 動作モデル | 自律エージェント | コマンド提案・補完 |
| コードの読み書き | ファイル直接編集 | エディタ経由 |
| コマンド実行 | 実行して結果を判断 | 提案のみ |
| コンテキスト範囲 | プロジェクト全体 | 現在のファイル・コマンド |

Copilot CLIは「次に打つコマンドを教えてくれる」ツールです。Claude Codeは「タスクを渡したら最後まで処理してくれる」ツールです。

この違いは自動化の深さというより、**誰が判断するか**の問題です。Copilotは提案して人間が判断・実行します。Claude Codeは判断と実行を自分でやります。

## 課金モデルの違いが効いてくる場面

GitHub Copilotはプロンプト単位の課金設計です。1回の質問に対して1回分消費されます。会話型の使い方（バグの相談、コードレビュー、設計の壁打ち）は10ドルで十分です。

問題になるのは自律実行を連続して使う場合です。「認証モジュールにテストを追加して、失敗したら修正して」という指示を出すと、ファイル読み取り・テスト生成・実行・エラー確認・修正・再実行と、複数のステップが連続します。Copilotはこのステップ数に応じてリクエストを消費します。

Claude Maxはトークン上限内なら何ターンでも使えます。1つのタスクで数十ターンの自律実行をしても追加課金はありません。

このリポジトリでは「textlint実行→修正→確認→commit」をまとめて任せることが多いので、ステップ数が積み上がります。この使い方では、プロンプト単位課金よりトークン上限課金の方が計算しやすいです。

## Copilot CLIが良い点

モデルの選択肢はCopilotの方が広いです。Claude Code（当然ながらClaudeしか使えません）と違い、CopilotはClaude系・GPT系・Gemini系から選べます。タスクに応じて使い分けたい場合は選択肢として有効です。

価格帯の幅広さも利点です。月10ドルから始められて、使い込んで足りなくなったらPro+（月39ドル）に上げる、という段階的な投資ができます。

## $190の差をどう考えるか

判断の基準は「判断を含む反復作業があるか」です。

手順が決まっていて途中で考えなくていい作業はスクリプトで十分です。途中の結果によって次の手順が変わる作業——実行→エラー確認→修正→再実行のループや複数ファイルをまたぐ変更——でこそ、Claude Codeの価値が出てきます。

組み込みの仕事ではこういう場面はあまりありません。ファイルをまたいだリファクタや、ツールチェーンを組み合わせた処理が多いWeb系の作業の方が恩恵を感じやすいです。

今のところClaude Codeを使い続けています。ただCopilotのエディタ補完はやめていません。コードを書くときの補完はCopilot、タスクをこなすときはClaude Codeという使い分けがしっくりきています。

$190の差は、補完と実行の差でもあり、提案と完結の差でもあります。
