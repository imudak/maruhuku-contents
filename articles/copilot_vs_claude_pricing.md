# 先日「Copilot で十分」と書いた私が、料金を詳しく調べて Claude に乗り換えることにした話

## 結論：Claude.ai Pro（月払い）をまず試してみます

先日、「Claude Codeが流行っているけど、VS Code + Copilotで困ってないならそのままでいい」という記事を書きました。

**しかし、今日料金を本気で計算したら、結論が180度変わりました。**

私の使用量（月2,500リクエスト以上）の場合、次のようになります。

- Copilot Free：月 $88、年間 $1,056
- Claude.ai Pro（月払い）：月 $20、年間 $240

**年間で約12万円（$816）の差**があります。

これは無視できない金額です。先日の記事では「使い勝手」だけで判断していましたが、**コストを無視していました**。今日、改めて自分の実際の支払い額を見て、計算し直したら、Claude.ai Proへの移行が圧倒的に合理的だという結論に至りました。

まずは**月払い（$20/月）で1-2ヶ月試して**、問題なければ年払い（$17/月）に切り替える予定です。

以下、詳しい料金体系の比較と、なぜこの結論に至ったかを説明します。

## はじめに

実際に使い続ける上で重要なのがコストです。
GitHub CopilotとClaudeの料金体系をまとめます。

この記事では、まず一般的な料金体系を整理した上で、実際に私がGitHub Copilotをヘビーに使っている実例を紹介します。

### 用語の整理

Anthropicが提供するClaudeには複数の利用方法があります。

- **Claude.ai** - Web/アプリでの利用（Free/$17-20/$100-200の3プラン）
- **Claude Code** - **ターミナルで動作する CLI ツール**。Pro/Maxプラン、またはAPIキーで利用可能
- **Claude API** - トークンベースの従量課金API（$3/$15 per 1M tokens）
- **VS Code での Claude 利用** - Claude APIを使う拡張機能経由（Claude Codeとは別物）

VS CodeでClaudeを使う場合、次の2つの方法があります。

1. **Claude.ai Pro/Max サブスクリプション** - ブラウザでのClaude.ai + Claude Code CLI + 優先アクセス
2. **Claude API を直接使用** - VS Code拡張機能経由で従量課金（月額固定費なし）

本記事では、VS Codeでの開発を前提に、GitHub CopilotとClaudeの各利用方法の料金を比較します。

## GitHub Copilot の料金体系

### 基本プラン

- Copilot Freeは $0/月（月300のPremium requestsが含まれる）
- Copilot Individualは $10/月（年払いで$100/年）
- Copilot Businessは $19/ユーザー/月
- Copilot Enterpriseは $39/ユーザー/月

### Premium Request SKUs（追加リクエスト）

GitHub Copilotでは、Claude Sonnet 4.5などの高性能モデルを使用する場合、Premium Request SKUsを購入します。

追加リクエストの料金体系は以下の通りです。

- 料金は約$10で、追加のプレミアムリクエスト枠を購入
- 対象モデルはClaude Sonnet 4.5、GPT-4o、その他プレミアムモデル
- 基本枠として月に一定回数のプレミアムリクエストが含まれている
- 基本枠を超えた場合、都度SKUを購入する形式

特徴は以下の通りです。

- 月額固定費と従量課金の組み合わせ（Freeプランは月300のPremium requests無料枠あり）
- プレミアムモデル（Claude Sonnet 4.5など）を使う場合、チャット・補完の両方でPremium requestsを消費
- 標準モデルでの補完やチャット機能は基本料金内で使い放題
- プレミアムリクエストの消費は、長い会話や複雑なタスクで増加

## Claude の料金体系

Claude.aiのサブスクリプションプランと、開発者向けのClaude APIがあります。

### Claude.ai プラン（Claude Code を含む）

**Freeプラン**（$0/月）

- 短いメッセージ：約40回/日（長いメッセージや添付ファイルがある場合は20〜30回/日）
- リセット：5時間ごと（ローリングウィンドウ方式）
- 利用可能モデル：Claude Sonnet 4.5、Haiku 3.5
- **Claude Code CLI は利用不可**（サブスクリプション認証の場合、Pro/Maxが必要）
- **ただし、Claude API キーを使えば全機能が利用可能**（従量課金）
- **Opus 4.5 は利用不可**
- Web、iOS、Android、デスクトップで利用可能
- 混雑時は優先度が下がり、応答が遅くなる可能性あり

**Pro プラン**：$17/月（年払い）または $20/月（月払い）

- メッセージ数：約45回/5時間（約216回/日）
- Claude Code CLI：週40〜80時間（Sonnet 4使用可能）
- 全モデルへのアクセス：Sonnet 4.5、**Opus 4.5**、Haiku 3.5
- リセット：5時間ごと（ローリングウィンドウ方式）
- コンテキストウィンドウ：200Kトークン
- 優先アクセス：混雑時でも安定
- 小規模コードベース向け
- **VS Code では**：Claude.aiの認証を使ってClaude API経由で利用（CLIとは別）

**Max プラン**：$100/月（5x）または $200/月（20x）

#### Max 5x（$100/月）の内容

- メッセージ数：約225回/5時間（Pro比5倍）
- Claude Code：週140〜280時間（Sonnet 4）+ 週15〜35時間（Opus 4）
- Opus 4使用時、20%到達で自動的にSonnet 4に切り替え

#### Max 20x（$200/月）の内容

- メッセージ数：約900回/5時間（Pro比20倍）
- Claude Code：週240〜480時間（Sonnet 4）+ 週24〜40時間（Opus 4）
- Opus 4使用時、50%到達で自動的にSonnet 4に切り替え
- 制限超過時、API料金で追加購入可能

共通機能は以下の通りです。

- メモリ機能（会話履歴を保持）
- 最高優先アクセス
- 大規模コードベース向け

重要なポイントは以下の通りです。

- **Free プランでは Claude Code が利用不可**（Proプラン以上が必要）
- Pro/Maxプランは使用枠が増えるだけで、機能自体は変わらない
- VS Code拡張機能とCLIは同じClaude Codeで、会話履歴を共有
- Claude（チャット）とClaude Code（コーディング）の使用量は**共有される**
- 2025年8月28日以降、週次制限が導入（パワーユーザー対策）

### Claude API（開発者向け従量課金）

Claude APIは**トークンベース**の従量課金制で、開発者がプログラムから直接利用するためのものです。

#### Claude Sonnet 4 (2025年1月時点の最新料金)

- 入力トークンは $3.00 / 1Mトークン
- 出力トークンは $15.00 / 1Mトークン
- キャッシュヒット（入力）は $0.30 / 1Mトークン
- キャッシュ書き込み（入力）は $3.75 / 1Mトークン

#### 参考: 他のモデル

- Claude Opus 4は入力 $15.00 / 出力 $75.00 / 1Mトークン
- Claude Haiku 4は入力 $0.25 / 出力 $1.25 / 1Mトークン

特徴は以下の通りです。

- 完全従量課金（使った分だけ支払い）
- 月額固定費なし
- プロンプトキャッシュを活用すると、繰り返し使う部分のコストが90%削減
- 会話の長さや複雑さに応じてコストが変動

#### プロンプトキャッシュとは

Claude APIでは、同じコンテキスト（例えばコードベース全体）を繰り返し送信する場合、2回目以降はキャッシュされた部分のコストが90%削減されます。

- 通常の入力トークン：$3.00 / 1Mトークン
- キャッシュヒット（入力）：$0.30 / 1Mトークン（90%削減）

例として、10万トークンのコードベースを毎回送信する場合を考えます。

- 1回目：$0.30（通常料金 + キャッシュ書き込み）
- 2回目以降：$0.03（キャッシュヒット）

同じコードベースで複数の会話をする場合、コストが10分の1になります。

#### VS Code での利用について

VS CodeでClaudeを使う場合、次の2つの選択肢があります。

1. **Claude.ai Pro/Max サブスクリプション（$17-200/月）**
   - Claude.ai Webインターフェース + Claude Code CLI + VS Codeでの利用が全て含まれる
   - VS Code拡張機能はClaude.aiの認証を使ってAPI経由でアクセス
   - 月額固定で使用量の上限が明確（45回/5時間など）

2. **Claude API を直接使用（従量課金）**
   - VS Code拡張機能にAPIキーを設定
   - 使った分だけ支払い（$3/$15 per 1M tokens）
   - FreeプランユーザーでもAPIキーがあれば全機能が使える
   - ライトユーザーの場合、サブスクリプションより安くなる可能性が高い

#### 選択の目安

- 月に数100回以上使う → サブスクリプション（定額で安心）
- 月に数10回程度 → API直接（使った分だけで安い）
- Claude Code CLIも使いたい → サブスクリプション必須

## 一般的な使い分けの目安

私の実例を踏まえて、一般的な使い分けの指針をまとめます。

### GitHub Copilotが向いている場合

#### 1. コード補完機能を重視する場合

- コード補完、インラインチャット、エディタ統合が充実
- 標準モデルでの補完は基本料金内で使い放題

#### 2. 使用量が少ない場合

- 月300リクエスト以内なら完全無料
- たまにしかプレミアムモデルを使わない場合に最適

#### 3. 都度課金が気にならない場合

- 使った分だけ支払うスタイルが好みの場合
- 月ごとの変動が大きい使い方をする場合

### Claudeが向いている場合

#### 1. 定額使い放題が良い場合（Claude.ai Pro/Max）

- Proプラン（$17-20/月）で約45回/5時間（1日約216回）使える
- Claude Code CLIも週40-80時間使える
- 使用量を気にせず作業できる
- 月に300リクエスト以上使う場合はこちらが有利
- コストが予測可能で予算管理しやすい
- 12月のような2,500リクエストの月でもProプランなら追加費用なし

#### 2. VS Code で Claude を使いたい場合（2つの方法）

方法1: Claude.ai Pro/Maxサブスクリプションです。

- VS Code拡張機能 + Claude Code CLI + Webインターフェースが全て使える
- 月額固定で使用量の上限が明確
- 月100回以上使う場合に有利

方法2: Claude API直接利用です。

- VS Code拡張機能にAPIキーを設定
- 使った分だけ支払い（従量課金）
- FreeプランでもAPIキーがあれば全機能が使える
- ライトユーザーの場合、サブスクリプションより安い可能性が高い
- Claude Code CLIは使えない

#### 3. 従量課金が良い場合（Claude API 直接利用）

- 月額固定費なし、使った分だけ支払い
- プロンプトキャッシュで繰り返し部分のコストが90%削減
- 月によって使用量が大きく変動する場合に有利

### ハイブリッド戦略

- 月額 $17-20の定額で使用量を気にせず作業できる

例3: Copilot Freeのみ（最もコスパ重視）です。

### 料金面での判断基準

各パターンに応じた選択肢は次の通りです。

- **月300リクエスト以内のライトユーザー**：Copilot Freeが最も安い（無料、VS CodeでAIアシスタントも使える）
  - またはClaude API直接利用（月$1-5程度、FreeプランでもAPIキーがあれば使える）
- **月300-1000リクエスト程度のミドルユーザー**：Claude.ai Pro $17-20/月が有利（約45回/5時間 + Claude Code CLI週40-80時間）
  - またはClaude API直接利用（月$10-30程度、使用量に応じて変動）
- **1日100回以上使うヘビーユーザー**：Claude.ai Max $100-200/月の使い放題が有利（Max 5x: 約225回/5時間、Max 20x: 約900回/5時間）
  - またはClaude API直接利用（月$50-150程度、プロンプトキャッシュで削減も）
- 予算の予測可能性を重視する場合：Claude.ai Pro/Maxのサブスクリプション（5時間ごとにリセット）
- 従量課金が良い場合：Claude API直接利用（使った分だけ、Freeプランでも使える）
- Claude Code CLIも使いたい場合：Claude.ai Pro以上が必須（API認証ではCLI不可）
どちらを選んでも、AIアシスタントを効果的に使うことで開発効率は大きく向上します。
