## はじめに — これは実験の記録です

こんにちは、**imudak**です。合同会社花丸福（まるふくこうぼう）という一人法人をやっています。

この連載は、**個人開発者が1年間で100パターンの収益化を試みる**というちょっと無謀な実験を、リアルタイムで記録していくものです。

大事なことを先に言っておくと、**この記事を書いている時点で、結果はまだ出ていません**。成功するかもしれないし、盛大にコケるかもしれない。でも、だからこそ記録する価値があると思っています。

個人開発の収益化記事って、成功した人が後から書くものがほとんどですよね。生存者バイアスの塊です。この連載では、**失敗も含めて全部オープンにします**。読者のみなさんには「他人の実験を追体験する」つもりで楽しんでもらえたら嬉しいです。

## 自己紹介

簡単にバックグラウンドを。

- **合同会社花丸福** 代表。一人法人です
- 本業はソフトウェアエンジニア（経験15年くらい）
- AI（主にClaude）を活用した開発が日常
- 個人開発歴はそこそこ長いが、**マネタイズは下手**

最後の一行が重要です。作るのは好きだけど、売るのが苦手。技術者あるあるですね。この連載は、そんな自分が「仕組みで殴る」アプローチで収益化に挑む記録です。

## なぜ100パターンなのか

「1つのプロダクトを磨き上げて成功させる」——これが王道だと思います。でも、個人開発の現実を見てください。

- Product Huntに出しても反応ゼロ
- 3ヶ月かけて作ったアプリのDAUが5人
- 「これは来る！」と思った機能に誰も興味がない

**当たりを見つけるには、数を打つしかない。**

これはスタートアップの世界では常識です。Y Combinatorの採択率は約1.5%。つまりプロの目利きでも、67個見て1個当たるかどうか。素人の個人開発者が「これだ！」と確信した1つのアイデアが当たる確率は、推して知るべしです。

だから100パターン。年間100個の収益化実験を回します。

「100個も作れるわけないだろ」と思いますよね。その通りです。普通にやったら無理です。だから**普通にやらない**。

## 3つの柱

この計画を支える仕組みは3つあります。

### 1. 事前検死（Pre-mortem）

実験を始める**前に**「これが失敗するとしたら、なぜか？」を考えます。

- 需要がない？→ 事前にキーワード調査とLP検証
- 技術的に無理？→ 2日でプロトタイプが作れないなら撤退
- 競合が強すぎる？→ ニッチを攻めるか、差別化ポイントを明確に

やらない理由を先に洗い出すことで、**始めてから「やっぱダメだった」を減らす**。当たり前のことですが、個人開発者は「作りたい」が先行して、ここをスキップしがちです（自戒）。

### 2. 運用コスト最小化

100個の実験を並行するには、1個あたりの運用コストを極限まで下げる必要があります。

- **サーバーレスファースト** — 使われなければ$0
- **静的サイト優先** — Cloudflare Pages / Vercel の無料枠
- **決済はStripe/Gumroad** — 自前で組まない
- **月間維持費の上限は1実験あたり$5** — 超えたら構成を見直す

「作って放置できる」状態を目指します。メンテに追われたら100個なんて回せません。

### 3. 実験トラッカー

すべての実験をNotionで一元管理します。各実験について記録するのは：

| 項目 | 内容 |
|------|------|
| 実験ID | P-001, P-002, ... |
| カテゴリ | アプリ / コンテンツ / テンプレ / SaaS / LP |
| ステータス | 企画中 → 開発中 → 公開 → 計測中 → 撤退 or 継続 |
| 投下時間 | 企画・開発・運用それぞれ |
| 収益 | 日次/月次 |
| 撤退判定日 | 14日後 / 30日後 |
| 学び | 何がわかったか |

数字で判断する。感情で続けない。これが大事です。

## 収益化の5カテゴリ

100パターンを5つのカテゴリに分類しています。

### 📱 アプリ（20本目標）
ウェブアプリ、Chrome拡張、モバイルアプリなど。フリーミアムか買い切りモデル。AIを使えば、ニッチなツールを短期間で作れる時代です。

### 📝 コンテンツ（25本目標）
Zenn有料記事、電子書籍、テンプレート付き解説。**この連載自体もここに含まれます**（メタですね）。コンテンツは在庫リスクゼロ、限界費用ゼロという最強の特性を持っています。

### 📋 テンプレート・素材（20本目標）
Notionテンプレート、Figmaテンプレート、プロンプト集など。「作業を省略したい人」に売る。単価は低いが、数を出しやすい。

### 🔧 ツール / SaaS（15本目標）
月額課金モデル。開発コストは高いが、当たればストック収益になる。MRR（月次経常収益）を積み上げる本丸。

### 🧪 LP検証（20本目標）
プロダクトを作る前に、LPだけ先に作ってウェイトリスト登録を募る。需要がなければ開発しない。**最もコスパの良い実験**です。

## 最大の課題：人間がボトルネック

ここまで読んで「計画はわかった。でも1人で年間100個は物理的に無理では？」と思ったあなた、正しいです。

1実験あたり平均3日（企画1日+開発1.5日+公開0.5日）としても、**300日分の工数**が必要です。土日フル稼働しても足りません。本業もあるし。

### 解決策：AIに任せる（Tier分類）

そこで、すべての実験を**AI自律度**で3つのTierに分けます。

| Tier | AI関与度 | 人間の役割 | 目標本数 |
|------|---------|-----------|---------|
| **Tier 1** | AI自律型 | 企画承認＋最終チェックのみ | 50本 |
| **Tier 2** | AI協働型 | 方針決定＋レビュー | 30本 |
| **Tier 3** | 人間主導型 | 全工程に関与 | 20本 |

**Tier 1が肝**です。企画書を承認したら、AIが調査・開発・デプロイ・LP作成まで自律的に実行する。人間は最終チェックだけ。

「それ、本当にできるの？」——正直、まだわかりません。だからこそ実験です。

### Tier 1 候補の一部

現時点でTier 1（AI自律型）として計画しているものをいくつか紹介します：

- **Notionテンプレート量産** — カテゴリ別にAIが設計・作成・出品
- **プロンプト集の販売** — 用途特化のプロンプトをパッケージ化
- **ニッチ情報まとめサイト** — 特定ジャンルの情報を自動収集・構造化
- **LP検証の自動化** — キーワード調査→LP生成→広告出稿→計測を一気通貫

これらは「人間の創造性」よりも「構造化された作業の実行力」が求められるタスクです。AIの得意分野ですね。

## 撤退基準の設計

100本ノックで一番大事なのは、実は**やめる基準**です。

ダメなものにしがみつくと、次の実験に進めません。サンクコスト（もう投資した時間）に引きずられて「もうちょっと頑張れば...」と思ってしまうのが人間です。だから、機械的に判断するルールを事前に決めます。

### 14日ルール（公開後14日）

以下のいずれかを満たさなければ**撤退**：

- PV 100以上（コンテンツ系）
- ダウンロード or 登録 10以上（アプリ/ツール系）
- ウェイトリスト登録 5以上（LP検証系）

### 30日ルール（公開後30日）

14日ルールをクリアした実験に対して：

- **収益 $1 以上**、または
- **明確な成長トレンド**（週次で数字が伸びている）

どちらもなければ撤退。

撤退 ≠ 失敗です。**「この方向では需要がない」という学びを得た**ということ。100本ノックにおいて、素早い撤退は最大の武器です。

## おわりに

正直に言うと、100本全部やりきれる自信はありません。途中で心が折れるかもしれないし、10本目くらいで「これ無理だわ」と気づくかもしれない。

でも、**やらない理由を探すより、やってみてダメだった理由を記録するほうが、100倍価値がある**と思っています。

この連載が、同じように「個人開発で稼ぎたいけど、何から始めればいいかわからない」という人の参考になれば嬉しいです。一緒に実験しましょう。
