## はじめに

個人開発アプリをGoogle Playに公開するまでの道のりを記録します。本記事では、Google Play Consoleのアカウント作成から、アプリ申請に必要な設定項目を順番に解説します。

対象アプリ：**gachi-todo**（目標特化型TODOアプリ / Flutter製）

## 前提条件

- Googleアカウント
- 登録料 $25（一度きり）
- 本人確認書類（日本の場合：運転免許証、マイナンバーカード等）

## 1. デベロッパーアカウント登録

### 1.1 Google Play Consoleにアクセス

[Google Play Console](https://play.google.com/console) にアクセスし、Googleアカウントでログイン。

### 1.2 アカウントの種類を選択

- **個人** または **組織** を選択
- 個人開発の場合は「個人」でOK

### 1.3 登録料の支払い

- $25（約3,700円）をクレジットカードで支払い
- 一度払えば永久に有効

### 1.4 本人確認

2023年以降、個人デベロッパーも本人確認が必須になりました。

**必要なもの：**
- 政府発行の身分証明書（運転免許証、マイナンバーカード等）
- 本人確認書類と自分の顔が映った写真

**確認プロセス：**
1. Jumioという認証サービスに転送される
2. 身分証明書の写真をアップロード
3. 顔写真を撮影（セルフィー）
4. 審査完了まで数日待つ

:::message
本人確認が完了するまで、アプリの公開はできません。早めに済ませておきましょう。
:::

## 2. アプリの作成

### 2.1 新しいアプリを作成

ダッシュボードから「アプリを作成」をクリック。

**入力項目：**
- **アプリ名**: gachi-todo
- **デフォルトの言語**: 日本語
- **アプリまたはゲーム**: アプリ
- **無料または有料**: 無料

### 2.2 申告事項

作成時に以下の確認が求められます：
- デベロッパープログラムポリシーへの同意
- 米国輸出法の遵守確認

## 3. アプリの設定を完了する

アプリ作成後、「アプリの設定を完了する」セクションに複数のタスクが表示されます。

### 3.1 プライバシーポリシーを設定する

**必須項目**。アプリがどのようなデータを収集・使用するかを説明するページのURLを登録します。

**プライバシーポリシーの作成方法：**

1. **Notionで作成**（おすすめ）
   - 無料で使える
   - 編集が簡単
   - Web公開機能でURLを取得可能

2. **GitHub Pagesで作成**
   - リポジトリがpublicなら無料
   - privateの場合はPro以上のプランが必要

**gachi-todoの例：**
```
https://maruhuku.notion.site/gachi-todo-privacy-policy
```

:::message
NotionでカスタムURLを設定するには、ページを公開後「リンクをカスタマイズ」から設定できます。日本語タイトルだとIDベースのURLになるため、英語のカスタムURLを設定するのがおすすめです。
:::

**プライバシーポリシーに含めるべき内容：**
- 収集するデータの種類
- データの使用目的
- 第三者への提供の有無
- データの保存場所
- 問い合わせ先

### 3.2 アプリのアクセス権

アプリが特別なアクセス権（ログインなど）を必要とするかを設定します。

**選択肢：**
- **すべての機能が制限なく利用可能** ← 一般的なアプリはこちら
- **すべてまたは一部の機能が制限されている** ← ログイン必須アプリなど

gachi-todoはローカル完結型なので「すべての機能が制限なく利用可能」を選択。

### 3.3 広告

アプリに広告が含まれるかを申告します。

- **広告あり**: AdMobなどを使用している場合
- **広告なし**: 広告を表示しない場合

gachi-todoは広告なしなので「いいえ、このアプリに広告は含まれていません」を選択。

### 3.4 コンテンツのレーティング

IARCの質問票に回答して、アプリの年齢レーティングを取得します。

#### カテゴリ選択

最初にアプリのカテゴリを選択します：
- **ゲーム** - ゲームアプリの場合
- **ソーシャルまたはコミュニケーション** - SNS、メッセージアプリ
- **その他のすべてのアプリの種類** - 上記以外

生産性アプリ（gachi-todo）は「**その他のすべてのアプリの種類**」を選択。

#### 質問票の回答例（生産性アプリの場合）

| 質問 | 回答 |
|------|------|
| ダウンロード済みアプリにレーティング関連コンテンツが含まれるか | いいえ |
| ユーザーコンテンツの共有機能があるか | いいえ |
| オンラインコンテンツ（AI生成含む）にアクセスするか | はい（AI機能がある場合） |
| 暴力的なテーマが含まれるか | いいえ |
| 性的コンテンツが含まれるか | いいえ |
| 不快な表現が含まれるか | いいえ |
| 規制物質への言及があるか | いいえ |
| 年齢制限商品の宣伝・販売をしているか | いいえ |
| 位置情報を他ユーザーと共有するか | いいえ |
| デジタル商品の購入ができるか | いいえ |
| 現金報酬・NFT等を発行するか | いいえ |
| ウェブブラウザ/検索エンジンか | いいえ |
| ニュース/教育製品か | いいえ |

:::message alert
AI機能（ChatGPT連携など）がある場合、「オンラインコンテンツ」で「はい」を選択する必要があります。AI生成コンテンツは明示的に例として挙げられています。
:::

### 3.5 ターゲットユーザー

アプリの対象年齢層を設定します。

**選択肢：**
- 5歳以下
- 6〜8歳
- 9〜12歳
- 13〜15歳
- 16〜17歳
- 18歳以上

:::message alert
13歳未満を対象に含める場合、COPPA（児童オンラインプライバシー保護法）への準拠が必要になり、審査が厳しくなります。子供向けに設計していないアプリは「**18歳以上**」を選択するのが無難です。
:::

「18歳以上」を選択すると、追加オプションとして「未成年による使用を制限する」チェックボックスが表示されますが、通常のアプリでは**チェック不要**です。

### 3.6 データセーフティ

2022年に導入された重要なセクション。アプリがどのようなデータを収集・共有するかを詳細に申告します。

#### 基本質問

| 質問 | gachi-todoの回答 |
|------|------------------|
| ユーザーデータを収集または共有するか | はい（AI機能がある場合） |
| 転送時にデータは暗号化されるか | はい（HTTPS通信） |
| アカウント作成方法 | 許可していない |
| アプリ外アカウントでログインできるか | いいえ |
| データ削除リクエスト方法を提供しているか | いいえ |

#### データの種類（AI機能がある場合）

AI審査機能など、外部APIにデータを送信する場合は申告が必要です。

**選択するカテゴリ：**
- アプリのアクティビティ > **その他のユーザー作成コンテンツ**

**詳細設定：**
| 項目 | 回答 |
|------|------|
| 収集か共有か | 共有（サードパーティAPIに送信するため） |
| 共有する理由 | アプリの機能（AI審査機能の実現のため） |

:::message
ローカル保存のみでサーバー通信しないアプリは「データを収集しない」で問題ありません。ただしオプションでもAI機能があるなら正直に申告しましょう。
:::

### 3.7 行政アプリ

政府機関のアプリかどうかを回答。一般的なアプリは「**いいえ**」。

### 3.8 金融取引機能

金融サービス（送金、投資、暗号通貨など）を提供するかを回答。TODOアプリは「**いいえ**」。

### 3.9 健康

医療・健康関連の機能を提供するかを回答。TODOアプリは「**いいえ**」。

## 4. アプリの構成と表示を管理する

### 4.1 アプリのカテゴリを選択し、連絡先情報を提供する

**カテゴリ選択：**
- **アプリケーション** > **仕事効率化** または **ライフスタイル**

**連絡先情報：**
- メールアドレス（必須・公開される）
- 電話番号（任意）
- ウェブサイト（任意）

### 4.2 ストアの掲載情報を設定する

アプリストアに表示される情報を設定します。

**必要な素材：**

| 項目 | サイズ | 必須 |
|------|--------|------|
| アプリアイコン | 512x512 PNG | ✅ |
| フィーチャーグラフィック | 1024x500 PNG/JPG | ✅ |
| スクリーンショット（スマートフォン） | 最低2枚 | ✅ |
| スクリーンショット（タブレット） | 最低1枚 | 推奨 |

**テキスト情報：**

| 項目 | 文字数制限 |
|------|------------|
| 簡単な説明 | 80文字 |
| 詳しい説明 | 4000文字 |

## 5. リリースの準備

### 5.1 AABファイルの準備

Google Playでは2021年8月以降、AAB（Android App Bundle）形式が必須です。

**Flutterでの生成：**
```bash
flutter build appbundle --release
```

生成場所：`build/app/outputs/bundle/release/app-release.aab`

### 5.2 署名キーの設定

初回リリース時にアップロードキーを設定します。Google Play App Signingを使用すると、キーの管理をGoogleに任せられます。

### 5.3 リリーストラック

| トラック | 用途 |
|----------|------|
| 内部テスト | 開発チーム向け（最大100人） |
| クローズドテスト | 限定ユーザー向けテスト |
| オープンテスト | 誰でも参加可能なテスト |
| 製品版 | 一般公開 |

**初回は「クローズドテスト」推奨：**
- 製品版公開には20人以上のテスターが14日間テストする必要がある（2023年11月以降の新規デベロッパー向け要件）

## 6. 審査と公開

### 6.1 審査プロセス

- 初回審査：数日〜1週間程度
- 自動審査と人間による審査の両方が行われる
- ポリシー違反があるとリジェクト（修正して再申請可能）

### 6.2 公開後

- ストアに反映されるまで数時間かかることがある
- 公開後もポリシー違反で削除される可能性あり

## まとめ

Google Play Consoleの設定は項目が多いですが、一つずつ進めれば難しくありません。

**重要ポイント：**
1. 本人確認は早めに済ませる
2. プライバシーポリシーは必須（Notionで簡単に作成可能）
3. コンテンツレーティングはカテゴリ選択に注意
4. AI機能があるならデータセーフティで正直に申告
5. 子供向けでないなら対象年齢は18歳以上が無難
6. 初回はクローズドテストから始める

次回の記事では、実際のAABアップロードからテスト公開までの流れを解説予定です。
